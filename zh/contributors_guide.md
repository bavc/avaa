## 基础知识 {#intro}

感谢您查阅贡献者指南！没有热心的影音爱好者社区，AVAA 无法持续运营。

本指南提供了通过 GitHub 或 Prose 进行贡献的简明步骤。如果内容不太好理解，建议先快速学习 git、GitHub 和 Markdown 语法（稍懂 HTML 亦有帮助）——这些都很容易上手。更多学习资源请参阅 <a href="{{ site.baseurl }}/zh/faq.html">常见问题</a> 页面。

大多数页面底部都有"在 GitHub 中编辑"或"在 Prose 中编辑"的链接。

视频示例可嵌入任意来源，但如果能将其上传至 [Internet Archive 的 AV Artifact Atlas 合集](https://archive.org/details/avartifactatlas?sort=-date)则更佳。

## 提交 GitHub Issue {#issues}

发现了拼写错误、内容错误或明显失实的描述？请[提交一个 Issue](https://github.com/bavc/avaa/issues)！

有新瑕疵条目的想法（或希望扩充现有条目），但还不熟悉 GitHub 编辑方式？请[提交一个 Issue](https://github.com/bavc/avaa/issues)！

遇到无法识别的瑕疵，希望听听社区意见？请[提交一个 Issue](https://github.com/bavc/avaa/issues)！

操作步骤：
- 点击瑕疵页面底部的链接
- 点击右上角的"New Issue"
- 在表单中描述您希望做出的修改
- 如需协助鉴定瑕疵，请在主题行以"Artifact"开头，如"Artifact: 画面出现奇怪条纹"
- 如有功能改进建议，请在主题行以"Wishlist"开头
